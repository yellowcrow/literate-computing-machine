ТЗ ДЛЯ КАНДИДАТА (Live coding, React): мини-Wordle (угадай слово)

ФОРМАТ
- React (JS или TS — как в шаблоне).
- Ввод только с физической клавиатуры.
- Секретное слово захардкожено.
- Логику повторов букв (дубликатов) НЕ делаем в основной части — это будет доп. задание.

--------------------------------------------
ОПИСАНИЕ ЗАДАЧИ
Нужно реализовать упрощённую игру Wordle.

Правила:
- Секретное слово (target) фиксированное, 5 букв, латиница, верхний регистр.
  Пример: target = "REACT".
- У пользователя есть 6 попыток.
- Пользователь вводит слово по буквам с клавиатуры:
  - A–Z добавляют букву в текущую строку
  - Backspace удаляет последнюю букву
  - Enter отправляет попытку (только если введено ровно 5 букв)

После отправки попытки нужно показать статусы букв:
- correct  (зелёный): буква совпадает и стоит на своём месте
- present  (жёлтый): буква есть в слове, но не на своём месте
- absent   (серый): буквы нет в слове

Упрощение: при оценке present/absent можно использовать простую проверку “буква есть в target” (без корректной обработки повторов букв).

--------------------------------------------
UI ТРЕБОВАНИЯ
- Отрисовать поле 6×5 (6 строк, 5 клеток).
- В текущей активной строке показывать вводимые буквы.
- После Enter строка фиксируется и становится не редактируемой.
- Сообщение “Not enough letters” при Enter, если введено меньше 5 букв.
- Кнопка Restart (сбрасывает игру).
- При победе (угадал слово) показать сообщение “You win” и заблокировать ввод.
- При поражении (использованы 6 попыток) показать “Game over. Word was REACT” и заблокировать ввод.

--------------------------------------------
МИНИМУМ (must have)
1) Поле 6×5 отображается.
2) Ввод с клавиатуры:
   - буквы добавляются до 5
   - Backspace удаляет
   - Enter отправляет только при длине 5
3) Оценка статусов по правилам (без повторов букв).
4) Переход на следующую попытку после Enter.
5) Win/Lose состояния + блокировка ввода.

--------------------------------------------
НОРМ (хороший уровень)
1) Чистая структура state (guesses/current/attempt/gameStatus).
2) Аккуратный обработчик keydown с cleanup в useEffect.
3) Restart корректно сбрасывает всё состояние.
4) Разделение на компоненты минимум:
   - WordleGame (state + логика)
   - Board / Row / Cell (рендер)

--------------------------------------------
МАКСИМУМ (доп. задания, если остаётся время)
Выбрать любые 1–3:
A) Подсветка клавиатуры (on-screen клавиатуру НЕ обязательно рисовать, но можно):
   - хранить лучший статус для каждой буквы (correct > present > absent).
B) Проверка слова по списку (маленький словарь):
   - если введённое слово не в списке → “Not in word list”, попытка не тратится.
C) Корректная обработка повторов букв в оценке (дубликаты).
D) Улучшения UX: подсветка активной строки, небольшие анимации.
E) Accessibility: aria-live для сообщений, управление фокусом.

--------------------------------------------
ПОДСКАЗКА ПО СТРУКТУРЕ ДАННЫХ (не обязательна)
- target: string
- guesses: Array<{ word: string, statuses: Array<"correct"|"present"|"absent"> }>
- current: string
- attemptIndex: number (0..5)
- gameStatus: "playing" | "won" | "lost"
- message: string | null

Готовность: к концу решения должна быть рабочая игра по минимуму.
